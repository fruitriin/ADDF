# 経験ファイル 参照健全性検証

スキル定義内の `.exp.md` 参照が、自スキル名と一致し、読み込み・書き込み両方の指示を含んでいるかを検証する。

## 背景

ADDF の経験活用は「クオートなし @メンションによる自動展開」ではなく、**バッククォート付きファイル名 +
手動 Read 指示**の規約で統一されている。@メンション展開はスキルファイル内では効かず CLAUDE.md 専用の機構であることが
実証済みで（`.claude/addf/knowhow/ADDF/claude-md-at-mention.md`）、現行スキル群にも `@` 形式の参照は1件も存在しない
（Plan 0044 実測）。本スキルは旧仕様（クオートされた @メンションの検出・修正 — 該当パターンが存在しないため
実質的に機能しない死んだロジックだった）を廃し、現行規約に沿った参照健全性チェックに再定義する。

## 引数
- **引数なし**: 全スキルを検証する
- `$ARGUMENTS`: 特定のスキル名（例: `addf-gui-test`）を指定して検証する

## 手順

### Phase 1: スキャン

1. `.claude/commands/` と `.claude/addf/optional/commands/` 配下の全 `.md`（`.exp.md` を除く）を一覧取得する
2. 各ファイル本文から `` `<name>.exp.md` `` 形式の参照を抽出する
   （見出し「## 経験の活用」を持つスキルと、手順内に読み/書き指示を埋め込むスキルの両方の書き方を対象にする）

### Phase 2: 判定

3. 参照が1つもないスキル → **経験活用の記述なし**として一覧化する（エラーではない。単純なスキルは持たなくてよい）
4. 参照はあるが自スキル名と不一致（例: `addf-dev.md` が `addf-release.exp.md` を参照）→ **自己参照ミス**として報告する
5. 参照はあるが「読み込み」指示（実行前に読む旨）または「書き込み」指示（実行後に追記する旨）の**どちらかが欠けている**
   → 片手落ちとして報告する
6. `@<name>.exp.md` 形式（実際には展開されない誤解を招く書式）が使われている → 修正候補として報告する

### Phase 3: 報告

7. 検証結果を一覧で報告する:
   ```
   ## 検証結果

   | スキル | 参照ファイル | 自己参照 | 読み | 書き | 状態 |
   |---|---|---|---|---|---|
   | addf-dev.md | addf-dev.exp.md | OK | OK | OK | 正常 |
   | addf-lint.md | — | — | — | — | 経験活用の記述なし |
   ```
8. 問題があれば修正案を提示する（自動修正はしない — 文言の再構成はスキル本文の意図を汲む必要があるため、
   人間またはメインエージェントの手直しに委ねる）

## エラーケース

- `.claude/commands/` が存在しない → 「commands ディレクトリが見つかりません」と報告
- `.exp.md` 参照が1つもないスキル → スキップせず「経験活用の記述なし」として一覧に含める
  （Plan 0044 の実測でカバレッジの可視化自体に価値があると判明したため）

## 経験の活用
- 実行前に `addf-experience.exp.md` が存在すれば読み、過去の経験を考慮する
- 実行後、新たな教訓があれば `addf-experience.exp.md` に追記する
