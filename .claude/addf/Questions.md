# Questions

ループ中のエージェントが「閾値割れ（確信が持てない）」と判断した事項を記録する非同期質問箱。
エージェントは止まらず質問だけ置いて次へ進み、オーナーは気が向いたタイミングで Answer を書く。
Answer が書かれたら、エージェントは次のセッション開始時（ブートシーケンス 1.5）に Plan へ反映し、該当質問を「回答済み」へ移す。

質問の書式は `.claude/addf/Questions.example.md` の「Q例」に従う。
このファイルはオーナーとの共有チャンネルのため、リポジトリにコミットされる。

- 投下日時: 2026-07-10（cron 経由 `/loop` 5サイクル目・サイクル内で選べる「未着手」が
  0040/0048 のみになった時点）
- 状況: 投機実施自体はオーナー承認済み（2026-07-06）。ただし Plan 0040 は外部 Go バイナリ
  （ccchain）のインストール・`.ccchain.conf` のルール作成・`settings.local.json` への
  PreToolUse(Bash) hook 配線を伴う。この hook は**このセッション自身の以降の Bash 実行を
  ゲートする**ため、誤ったルールを書くと以降の cron サイクル（本ループ自身）が Bash を
  使えなくなり、手動介入なしに復旧できないリスクがある。Plan 自身も「単独投機・並列なし」を
  明記しており、cron 経由の並行実行（`.claude/addf/knowhow/ADDF/cron-loop-worktree-race.md`
  で実測済みの並行衝突リスク）と相性が悪い
- 質問: (a) このまま cron ループ内で `/addf-speculate` 経由で着手してよい（worktree 隔離下なので
  本体には影響しない前提で進める）/ (b) Plan 0037 の前例（Q2）同様、オーナー同席の別セッションで
  実施したい / (c) 他の判断
- 仮の判断: 静観（着手しない）。理由: worktree 隔離下でも「本体の hook 配線」自体は最終的に
  main へ統合する前提の Plan であり、統合後の検証を無人ループで行うのは避けたい
