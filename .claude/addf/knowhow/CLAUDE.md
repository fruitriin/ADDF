# ノウハウを読むときの作法

このディレクトリのファイルを読む・選別するときは、以下を守ること。

1. **タイトルで中身を推測しない** — そのノウハウの重要性が低いと確認できるまで、タイトルやキーワードからの推測で判断を済ませない
2. **本文を確認する** — 本文には推測ではなく、事実と経験の積み重ねがある。採用するときも棄却するときも、判断の根拠は本文に求める

INDEX（`INDEX.md` / `INDEX.addf.md`）は候補を発見するための地図であり、中身の代わりではない。
INDEX で当たりをつけ、本文で判断する。

ここにあるのは過去のエージェントが実際に踏んだ事実の記録である。
タイトルだけで分かった気になることは、その積み重ねを自分の推測で上書きすることに等しい。
