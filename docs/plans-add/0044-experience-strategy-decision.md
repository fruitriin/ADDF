## 目的

`.exp.md`（経験ファイル）の運用方式について、2つの案（案A: スキル本文と分離した現行方式・案B: スキル本文に埋め込む一体化方式）のどちらを正式採用するかを決定する。

## 現状の挙動

- `.exp.md` はスキル本文（`.md`）と分離した個別ファイル。gitignore 対象で、ローカル環境で蓄積される
- スキル起動時に `<スキル名>.exp` を明示的に読ませる運用（現行 addf-* スキル群）
- 課題感（Plan 0026 起票時点）: 分離しているためスキル本文の変更と exp の変更が同期しにくい・オーナーの目に触れにくい・PR レビュー対象外

## 変更内容（項目）

### 項目1: 案A/案B の実測比較

- **案A（現行分離方式）** の運用実績: 直近サイクル群で `.exp.md` は実際にどう使われたか（更新頻度・スキル本文との乖離が問題を起こした事例）
- **案B（本文埋め込み方式）** の想定コスト: スキル本文サイズの増加・PR レビュー時の diff ノイズ・機械同期の複雑化
- 実測データを集めた上で判断する（現状「悩み」だけで進んでいない）

### 項目2: 決定と移行手順

- 案A 継続の場合: 現行維持で決定を明記して終わり
- 案B に切替の場合: 移行手順（既存 .exp.md → スキル本文の「経験」セクションへ統合）を書く
- ハイブリッド案があれば併記

## 影響範囲

- 案A 決定 → 変更なし
- 案B 決定 → 全 addf-* スキルの `.md` と `.exp.md` の統合作業
